## ⚡ 2. 深度速度优化历史 (ASR Speed Hacks)

为了将听写响应从最初的 **6~8 秒** 压缩至如今的 **~2 秒（光标仅闪烁 4 下）** 的绝对极限，系统实施了以下三项底层优化：

1. **设备快速开启与动态物理麦克风路由（Bypass 虚拟声卡，提速 ~500ms）**：
   * *问题*：安装了如 `iShotAudioPlugin`、`BlackHole` 等虚拟声卡后，静态指定 `none:default` 极易匹配到虚拟音频通道，录音内容会是纯静音。且默认调用 `ffmpeg` 会触发“连续互通相机”无线尝试唤醒 iPhone，卡顿 500ms+。
   * *解决*：在 `dictate_toggle.sh` 中加入 `get_mic_index` 音频分析。在录音前执行 `ffmpeg -list_devices` 动态分析当前设备，自动避开所有虚拟声道（匹配 `ishot|blackhole|loopback|zoom|soundflower|virtual` 等黑名单），精准锁定真实物理麦克风的底层索引号（如内置麦克风 `1`），动态路由至 `none:$mic_idx`，在 20ms 级瞬时开启的同时完美解决静音输入死穴。

2. **音频 5.5 倍压缩（提速网传时间）**：
   * *优化*：锁定 `ffmpeg -ar 16000 -ac 1`。强制以符合 ASR 规格的**单声道、16kHz 采样率**录制。音频体积缩小 5.5 倍（每秒仅 32KB），使录音的上传与 API 服务端解码时间成倍缩短。

3. **常驻长连接（免除 TLS 握手与 Python 启动，提速 1 秒+）**：
   * *优化*：引入 `dictate_daemon.py` 后台监听。在“开始录音”时自动探测并静默拉起它。利用用户说话的时间，在后台完成 Python 启动和 **Gemini API 的 HTTPS TLS 握手建立**。
   * 结束录音时，通过本地 **3ms 的 `curl` 请求** 直连守护进程，复用已经温好的 HTTPS 连接，彻底免去了建立加密通道的巨大网络开销。

4. **静音容错与 NoneType 防御（静音运行稳定性优化）**：
   * *优化*：针对静音或未开口情况，Gemini API 返回的空文本由 Python 脚本进行安全校验（使用 `response.text or ""` 防御空指针崩溃），并编写了针对静音播放器时间戳标志（如 `[ 0m0s - 0m3s ]`）的正则过滤规则，实现静音输入下优雅不崩溃，提升 100% 健壮性。

5. **硬件级粘贴按键与 macOS 无障碍权限主动提示（输入兼容性优化）**：
   * *优化*：将 AppleScript 的 `keystroke "v"` 彻底升级为硬件级虚拟按键码 `key code 9 using {command down}`，完全避免 Caps Lock 大写锁定键及任何中文输入法拼写状态对自动粘贴的干扰。
   * *权限引导*：一旦由于系统【辅助功能】权限不足导致模拟按键被 macOS 安全拦截，脚本会主动捕获错误，智能弹出系统提示框说明“转写已成功且文字已入剪贴板（可以直接 Cmd+V 粘贴）”，并手把手引导用户在「系统设置 -> 隐私与安全性 -> 辅助功能」中为快捷指令开启授权，实现完美闭环。

4. **静音容错与 NoneType 防御（静音运行提速与稳定性）**：
   * *优化*：针对静音或未开口情况，Gemini API 返回的空文本由 Python 脚本在底层进行安全校验（使用 `response.text or ""` 防御空指针崩溃），并编写了针对静音播放器时间戳标志（如 `[ 0m0s - 0m3s ]`）的正则过滤规则，实现静音输入下优雅不崩溃，提升 100% 健壮性。

## 🛠️ 3. 常见维护与规则自定义

### 📝 如何更新听写提示词（Prompt / 规矩）
如果您在未来想要添加新的听写规则、专有名词映射，或遇到特定的同音字纠错：
1. 打开 `/Users/yuanjin/localbin/dictate_daemon.py`（和备份的 `dictate.py`）。
2. 修改代码中大段的 `prompt = """ ... """` 字符串，在里面直接用自然语言追加规则。
3. 保存文件。
4. 在终端中运行以下命令，强制结束当前的常驻进程：
   ```bash
   pkill -f dictate_daemon.py
   ```
5. **无需手动重启**：当您下一次按下听写快捷键 `Control+Option+G` 时，控制脚本会检测到进程不在，并**自动静默拉起载入了您最新规则的守护进程**。

### 🎵 如何更换提示音效
1. 打开 `/Users/yuanjin/localbin/dictate_toggle.sh`。
2. 找到 `afplay /System/Library/Sounds/Ping.aiff &`（开始音）和 `afplay /System/Library/Sounds/Glass.aiff &`（结束音）。
3. 将文件名更换为 `/System/Library/Sounds/` 下的其它音频文件（如 `Bottle.aiff`）。保存后立即生效。
