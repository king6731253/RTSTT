我需要你在 macOS 上为我开发一个名为 RTSTT (Real-Time Speech to Text) 的全局语音听写系统。系统由快捷键 Control+Command+G 触发，按一次开始录音，再按一次结束录音，并将听写的文字自动粘贴 to 当前活跃的输入框中。为了达到极致的“秒级响应”体验，整个项目必须在底层进行如下深度速度优化：

1. **设备快速开启与动态麦克风路由（Bypass 虚拟声卡）**：在调用 ffmpeg 进行 avfoundation 录音时，先动态检测并剔除 iShotAudioPlugin、BlackHole 等虚拟声卡设备，精准自动匹配物理麦克风的索引号（如内置麦克风），并使用 "none:$mic_idx" 进行录音。这不仅彻底避免了 Continuity Camera 以及摄像头扫描的数百毫秒初始化卡顿，还完美杜绝了因匹配到虚拟声卡录音通道而导致录制到纯静音、提示“没有听到说话”的硬件路由故障。
2. **音频数据极致压缩**：录音时强制锁定为符合 ASR 听写规格的单声道（1 channel）和 16000Hz 采样率. 文件大小相较立体声缩小 5.5 倍，极大缩短音频的上传与 API 解析时间。
3. **常驻保活长连接（Warm-Connection Daemon）**：编写一个名为 `dictate_daemon.py` 的常驻后台守护进程（监听本机 127.0.0.1:18888）。在用户“开始录音”时，静默探测并唤醒该服务。利用用户说话的时间，在后台加载 Python 并在内存中与 Google Gemini API 服务器打通 HTTPS (Keep-Alive) 连接池。结束录音时，通过 3ms 的 curl 环回请求直连守护进程，直接省去 TLS 握手等 1.5 秒网络开销！
4. **一键容灾兜底机制**：控制脚本在本地 curl 调用出错或超时时，必须无缝且完全透明地回退调用独立的单次转写脚本 `dictate.py`，确保 STT 100% 成功。
5. **原汁原味粤语听写词库与 Prompt**：提示词要强行注入高频粤语口语字（哋、喺、咗、嘢、唔、乜、冇 等）的特征规范，严禁 AI 对粤语口语进行翻译、润色或调整语序，严格做到“字字对应，所听即所得”。同时，必须将规则写入 system_instruction 中，绝对杜绝当录音过短或静音时 AI 在输入框打出听写规则的问题。
6. **零延迟声效反馈**：完全剔除臃肿的 Tkinter UI 悬浮窗，改用异步执行的原生 afplay。开始时播放 Ping.aiff 音效，结束时播放 Glass.aiff 音效。
7. **强健的 NoneType 与静音响应过滤**：重构音频听写提取逻辑，采用安全语法 response.text or "" 防御 NoneType 崩溃异常；同时使用正则匹配（如 `^[\s\d\-ms\[\]\.:]+$`）强力拦截仅含时间戳的无意义静音响应（如 `[ 0m0s - 0m3s ]`），自动视为空字符串返回，实现 100% 稳定运行。
8. **硬件级键盘模拟与无障碍权限捕获**：模拟按键粘贴彻底重构为硬件级虚拟键码 `key code 9 using {command down}`，完全免疫 Caps Lock 大写锁定键及任何第三方中文输入法状态的干扰。同时，主动捕捉 macOS 辅助功能（Accessibility）模拟按键被拦截的权限报错（如系统错误 1002），弹出精美的中文引导弹窗，告知用户文字已复制到剪贴板中（可直接 Cmd+V 粘贴），并手把手引导用户在「系统设置 -> 隐私与安全性 -> 辅助功能」中为宿主程序（如快捷指令）开启控制权限，实现极致人性化的体验。
9. **多声道与窗口快捷键冲突防御**：系统不仅要在硬件上过滤 Continuity Camera 扫描和虚拟声卡，还必须在文档及指引中明确说明如何解决与 Magnet 或 Rectangle 等第三方窗口管理软件全局热键的冲突问题（通过禁用窗口缩放对应冲突快捷键或在快捷指令中无缝更换绑定为 `Control + Option + S`），提供完美的周边应用生态兼容性。
